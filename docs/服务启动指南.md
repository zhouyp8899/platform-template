# 服务启动指南

本文档说明如何启动Nacos、Gateway和Auth-Service，并解决服务发现问题。

## 前置条件

1. **启动Nacos服务**
   ```bash
   # 下载Nacos
   wget https://github.com/alibaba/nacos/releases/download/2.2.3/nacos-server-2.2.3.zip
   unzip nacos-server-2.2.3.zip
   cd nacos/bin

   # 启动Nacos（单机模式）
   ./startup.sh -m standalone

   # Windows
   startup.cmd -m standalone
   ```

2. **安装并启动MySQL**
   ```sql
   CREATE DATABASE platform IF NOT EXISTS DEFAULT CHARACTER SET utf8mb4;
   ```

3. **安装并启动Redis**
   ```bash
   # Windows启动Redis
   redis-server.exe redis.windows.conf
   ```

## 环境变量配置

在启动服务前，配置以下环境变量：

### 方式1：命令行参数

```bash
# 设置Nacos地址
export NACOS_ADDR=127.0.0.1:8848
export NACOS_USER=nacos
export NACOS_PWD=nacos

# 设置MySQL连接
export mysql.url=127.0.0.1:3306
export mysql.username=root
export mysql.password=your_password

# 设置Redis连接
export redis.host=127.0.0.1
export redis.port=6379
export redis.password=
```

### 方式2：系统环境变量

在Windows系统环境变量中添加：

- `NACOS_ADDR=127.0.0.1:8848`
- `NACOS_USER=nacos`
- `NACOS_PWD=nacos`
- `mysql.url=127.0.0.1:3306`
- `mysql.username=root`
- `mysql.password=your_password`
- `redis.host=127.0.0.1`
- `redis.port=6379`

## 常见问题

### 1. SERVICE_UNAVAILABLE错误

**错误信息：**

```
503 SERVICE_UNAVAILABLE "Unable to find instance for platform-auth-service"
```

**解决方案：**

#### 检查项1：服务是否注册到Nacos

访问Nacos控制台，确认服务已注册。

#### 检查项2：路由配置是否正确

确认`application-gateway.yml`中路由配置：

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: auth-service-route
          uri: lb://platform-auth-service  # 注意lb://前缀
          predicates:
            - Path=/api/admin/**,/api/auth/**
```

#### 检查项3：服务名称是否一致

确认：

- Auth-Service配置：`spring.application.name=platform-auth-service`
- Gateway路由：`uri: lb://platform-auth-service`

#### 检查项4：网络连通性

```bash
# 测试Nacos连接
curl http://127.0.0.1:8848/nacos/v1/ns/instance/list?serviceName=platform-auth-service

# 测试服务端口
curl http://localhost:7200/actuator/health
```

### 2. 连接拒绝错误

**错误信息：**

```
Connection refused: 127.0.0.1:8848
```

**解决方案：**

- 确认Nacos服务已启动
- 检查端口占用情况：`netstat -an | grep 8848`

### 3. MySQL连接错误

**错误信息：**

```
Communications link failure
```

**解决方案：**

- 确认MySQL服务已启动
- 检查数据库连接参数
- 确认数据库已创建：`CREATE DATABASE platform;`

### 4. Redis连接错误

**错误信息：**

```
Unable to connect to Redis
```

**解决方案：**

- 确认Redis服务已启动
- 检查Redis端口：默认6379
- 检查Redis密码配置

## 生产部署

### Docker部署

```dockerfile
# Auth-Service Dockerfile
FROM openjdk:17-slim
COPY build/libs/*.jar app.jar
ENTRYPOINT ["java","-jar","/app.jar"]
```

### Kubernetes部署

```yaml
# deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: platform-auth-service
spec:
  replicas: 3
  selector:
    matchLabels:
      app: platform-auth-service
  template:
    metadata:
      labels:
        app: platform-auth-service
    spec:
      containers:
      - name: auth-service
        image: platform-auth-service:latest
        ports:
        - containerPort: 7200
        env:
        - name: NACOS_ADDR
          value: "nacos-server:8848"
```

## 总结

启动顺序：

1. 启动基础服务（MySQL、Redis、Nacos）
2. 启动Auth-Service
3. 启动Gateway
4. 验证服务注册
5. 测试MDC链路追踪

关键配置：

- ✅ 服务名称一致性
- ✅ 路由配置正确
- ✅ 环境变量设置
- ✅ 网络连通性
